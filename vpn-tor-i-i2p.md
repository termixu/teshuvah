# 🔐 VPN, Tor и I2P

### 📌 Зачем нужны эти технологии

| Цель                             | VPN | Tor         | I2P         |
| -------------------------------- | --- | ----------- | ----------- |
| Скрытие IP-адреса                | ✅   | ✅           | ✅           |
| Защита трафика от провайдера     | ✅   | ✅           | ✅           |
| Доступ к заблокированным сайтам  | ✅   | ✅           | ⚠️ частично |
| Глубокая анонимность             | ❌   | ✅           | ✅           |
| Подходит для обычного браузинга  | ✅   | ⚠️ медленно | ⚠️ сложнее  |
| Открытые децентрализованные сети | ❌   | ⚠️ частично | ✅ полностью |

***

### 🔒 VPN — виртуальная частная сеть

**✅ Преимущества:**

* Шифрует трафик между устройством и сервером
* Подменяет IP-адрес
* Защищает в публичных сетях (кафе, Wi-Fi и т.п.)
* Простой в использовании

**⚠️ Недостатки:**

* Требует доверия к VPN-провайдеру
* Большинство платных и централизованы
* Некоторые ведут логи (см. **no-logs policy**)

**🛠 Рекомендованные VPN:**

| Название        | Тип             | Примечание                                    |
| --------------- | --------------- | --------------------------------------------- |
| **Mullvad**     | платный         | Без почты, без логов, можно оплатить наличкой |
| **Proton VPN**  | есть бесплатный | Швейцария, прозрачная политика                |
| **Outline VPN** | сам себе        | Запускается на VPS, контроль у тебя           |
| **WireGuard**   | протокол        | Лёгкий, быстрый, можно самому поднять сервер  |

**📥 Самостоятельный VPN:**

* Подними сервер на VPS (желательно в другой юрисдикции)
* Установи **WireGuard** или **Outline**
* Используй только через свой DNS

***

### 🧅 Tor — луковая маршрутизация

**🔐 Как работает:**

* Трафик шифруется в 3 слоя (луковица)
* Передаётся через 3 случайных узла
* Выходной узел видит только конечный запрос, но не отправителя

**✅ Преимущества:**

* Анонимный серфинг
* .onion-сайты (скрытые сервисы)
* Без регистрации и подписок

**⚠️ Ограничения:**

* Медленно (из-за нескольких прыжков)
* Не подходит для heavy traffic (стриминг, видео)
* Выходные узлы могут быть скомпрометированы (не вводить пароли без HTTPS!)

**📥 Инструменты:**

* [Tor Browser](https://www.torproject.org/) — для безопасного доступа
* `torsocks` — для перенаправления трафика CLI-программ
* `.onion` сайты — только внутри сети Tor

***

### 🕳️ I2P — невидимая сеть (Invisible Internet Project)

**⚙️ Как работает:**

* Шифрует трафик полностью внутри сети I2P
* Децентрализованная, P2P структура
* Нет единой точки отказа

**✅ Плюсы:**

* Надёжная анонимность для P2P и общения
* Поддержка блогов, email, чатов внутри сети
* Локальные сайты (`.b32.i2p`) работают без выхода в обычный интернет

**⚠️ Минусы:**

* Требует настройки и знания
* Тяжело интегрируется с обычным вебом
* Скорость невысокая

**📥 Установка:**

* [I2P](https://geti2p.net/) — Java-клиент или I2Pd (на C++)
* Запускаешь локальный роутер (127.0.0.1:7657)
* Доступ через прокси (127.0.0.1:4444)

***

### 🛠 Рекомендованные практики

* Никогда не используйте VPN + Tor вперемешку без понимания → можно **ослабить анонимность**
* Не логиньтесь в свои аккаунты через Tor/I2P
* Проверяйте **IP-адрес** на [ipleak.net](https://ipleak.net/)
* Используйте **изолированный браузер** для каждого инструмента
