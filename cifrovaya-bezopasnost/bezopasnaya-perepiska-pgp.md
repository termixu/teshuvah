# 💬 Безопасная переписка (PGP)

> 📌 PGP (Pretty Good Privacy) — это метод **шифрования и цифровой подписи писем**, который позволяет:
>
> * Защитить свои сообщения от посторонних глаз;
> * Доказать, что письмо отправлено именно вами;
> * Проверить подлинность полученного сообщения.

***

### 🔐 Что такое PGP?

PGP использует **асимметричное шифрование**:

* У тебя есть **открытый ключ** (для других — чтобы тебе шифровать);
* И **приватный ключ** (хранишь в секрете — чтобы расшифровывать и подписывать).

📬 Только ты можешь прочитать сообщение, зашифрованное твоим открытым ключом.

***

### 🛠 Где использовать PGP?

| Сценарий                             | Использование PGP                  |
| ------------------------------------ | ---------------------------------- |
| Переписка с единомышленниками        | Шифруй письма и прикрепления       |
| Подтверждение цифровой подписью      | Подписывай важные письма           |
| Проверка целостности файлов/прошивок | Подпись = доверие                  |
| Жизнь вне системы                    | Альтернатива Gmail, Telegram и пр. |

***

### ⚙️ Инструменты

**📬 Почтовые клиенты с PGP:**

* **Thunderbird** (встроенная поддержка PGP, Linux/macOS/Windows)
* **Claws Mail**, **Mutt** — для любителей минимализма и CLI
* **Mailvelope** — расширение для браузера (если нет другого варианта)

**🔐 Управление ключами:**

* **GnuPG** — основной инструмент (CLI, встроен в Linux)
* **Kleopatra** — графическая оболочка для GnuPG (Windows)
* **Seahorse** — для GNOME/ElementaryOS

***

### 🧱 Генерация ключей

```bash
gpg --full-generate-key
```

Выбери:

* RSA (или RSA + RSA)
* Размер 4096 бит
* Срок действия — по желанию (или бессрочно)
* Укажи имя (можно псевдоним) и почту

🔒 После создания ключа — обязательно создай **ревокационный сертификат**:

```bash
gpg --output revoke.asc --gen-revoke your@email.com
```

***

### 📤 Отправка открытого ключа

* Отправь свой `.asc` файл получателю вручную;
* Или опубликуй на [keys.openpgp.org](https://keys.openpgp.org/) (только если не боишься деанонимизации);
* Или пересылай напрямую при первом сообщении.

***

### 📩 Как зашифровать письмо

```bash
gpg -e -r email@example.com message.txt
```

* `-e` — encrypt (зашифровать)
* `-r` — получатель
* `message.txt` — файл с сообщением

💬 Также можно использовать встроенные кнопки в Thunderbird или другой почтовой программе.

***

### 🖊 Как подписать сообщение

```bash
gpg -s message.txt
```

Подпись доказывает, что сообщение — твоё.

***

### 🧪 Как проверить подпись

```bash
gpg --verify message.txt.sig message.txt
```

Если подпись действительна — ты уверен в авторстве.

***

### 🔐 Практика паломника

* Используй **отдельную автономную почту**, не связанную с обычной жизнью (например, Tutanota, ProtonMail, Disroot).
* Не светишь публично свой PGP-ключ — делишься только лично, через Tor или офлайн.
* Храни закрытый ключ **вне сети** (флешка, бумажная копия, QR на бумаге).
* Привязывай PGP к образу нового пути, не к старой личности.
